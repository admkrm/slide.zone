# Informatietechnologie 1 ## Introductie tot HTML ### Elementen voor opmaak en structuur #### Kristof Michiels
## In deze presentatie - Structuur en betekenis van een document - Welke elementen voorziet HTML voor ons? - Hoe maak je een keuze uit deze elementen? - Aanloop naar onze eerste oefeningen
## Goed geschreven HTML geeft structuur en betekenis aan je documenten
## Enkele termen - De content: de informatie in je documenten (teksten, afbeeldingen, links) - De Structuur: de outline van je document - De semantiek: de betekenis van iets. HTML-element zorgen voor semantiek/betekenis in een document - De HTML5 standaard: de versie van HTML5 die we vandaag gebruiken. Deze standaard wordt opgesteld door het W3C (www.w3. org/TR/html5/). De laatste voorstelde versie is HTML 5.2 (www.w3. org/TR/html52/).
## Eerst dit - Alle informatie binnen een document moet in een html-element verpakt zitten. Wordt naakte of anonieme tekst genoemd en zorgt ervoor dat het document als niet-correct wordt beschouwd - Heel belangrijk je elementen juist te kiezen => semantisch juist, op een manier die zo goed mogelijk aansluit bij de betekenis. - Hoe het er uit zal zien is hierbij niet van belang => je verandert dit met een stylesheet. - Semantisch correcte markup => verzekert ons dat het er goed uitziet op alle schermen, dat Google het juist kan inschatten, schermlezers ,...
## Drie content-elementen: paragrafen, hoofdingen en lijsten
## Paragrafen: <p> - Paragrafen zijn zowat de belangrijkste bouwstenen van een tekstdocument - Het element begint met de <p>-tag en eindigt met de </p>-tag - Paragrafen kunnen behalve tekst ook andere (inline) elementen bevatten (bvb. afbeeldingen) - Hoofdingen, lijsten of andere block-level elementen mogen evenwel niet ```

## Hoofdingen: <h1>, <h2>, ... , <h6> - Met hoofdingen geef je titels en subtitels weer. Belangrijke elementen om structuur aan te brengen - Het element begint met de <h1>-tag (bij een h1-element) en eindigt met de </h1>-tag - HTML geeft ons 6 niveau's: <h1> voor de belangrijkste hoofding. Van daaruit werk je verder met <h2>, <h3>... om tot een heldere documentstructuur en outline komen ```

Je belangrijkste titel

Een net iets minder belangrijke titel

Een ondertitel van het derde niveau

Een ondertitel van het vierde niveau

Een ondertitel van het vijfde niveau
Een ondertitel van het zesde niveau
```
## Regels rond hoofdingen - Op elke pagina mag slechts één <h1>-element voorkomen - De overige hoofdingen: zo vaak je wenst. Je hoeft ze dus niet alle zes te gebruiken - De structuur die je aanbrengt is zoals bij een boek: één titel (<h1>), hoofdstukken (allen <h2>), onderdelen van hoofdstukken (allen <h3>) enz... - Dus niet van <h1> tot <h6> voor elke volgende hoofding. Denk in niveau's ```

Mijn blog

Mijn eerste post

Blogtekst komt hier.

Mijn tweede post

Blogtekst komt hier.

```
## Oplijstingen: <ul> en <ol> - Wordt heel vaak gebruikt (links, afbeeldingen,opsommingen...) - Voor alles wat meer dan één keer voorkomt kan je een lijst gebruiken - HTML geeft ons verschillende manieren om lijsten te maken: we zien er vandaag twee
## Niet-geordende lijsten: <ul> - Voor een opsomming zonder volgorde gebruiken we het ul-element - Het element begint met de <ul>-tag en eindigt met de </ul>-tag - Elke lijst bestaat uit meerdere lijstelementen. Hiervoor gebruiken we li-elementen (li staat voor list items) - De browser plaatst standaard een bolletje voor elk list item, maar dit kan worden aangepast ```
  • Helvetica
  • Garamond
  • Times New Roman
  • Courier
  • Comic Sans
```
## Niet-geordende lijsten: <ul> - List items zijn de enige elementen die direct binnen een ul mogen gebruikt worden. Binnen een li ben je vrij - We noemen de li's de "kinderen" van het ul-element. Gebruik steeds insprongen voor betere leesbaarheid ```

  • Een titel

    Quisque rhoncus euismod pulvinar. Nulla non arcu at lectus.

  • Een andere titel

    Quisque rhoncus euismod pulvinar. Nulla non arcu at lectus.

```
## Geordende lijsten: <ol> - Worden gebruikt voor lijsten waarin de volgorde wel belangrijk is - Voor de rest werken ze op gelijkaardige manier als niet-geordende lijsten - De browser kent standaard een nummering toe. Ook dit kan aangepast worden via CSS ```
  1. Helvetica
  2. Garamond
  3. Times New Roman
  4. Courier
  5. Comic Sans
```
## Geordende lijsten: <ol> - Met start-attribuut - indien aanwezig - kan je de nummering aanpassen - We zagen het reeds bij het img-element in het algemeen voorbeeld: een element kan attributen of eigenschappen hebben. Soms verplicht, soms vrijblijvend ```
  1. Helvetica
  2. Garamond
  3. Times New Roman
  4. Courier
  5. Comic Sans
```
## De list-style-type eigenschap - Hiermee lopen we even vooruit: CSS geeft je controle over de manier waarop lijsten worden voorgesteld - De bullet kan worden weggelaten, een vierkant of open cirkel worden, ... - Bij geordende lijsten kan gekozen worden voor Romeinse cijfers, letters, ... ``` ul { list-style-type: disc; } ol { list-style-type: lower-roman; } ```
## Lijsten in elkaar vlechten - We kunnen lijsten in elkaar vervlechten. De browser zal automatisch inspringen en de bullets aanpassen - Je ziet: html schrijven is je hoofd erbij houden. Denk aan de insprongen ```

## Inline elementen
## Inline elementen - block level elementen: alle elementen die we tot nu toe hebben gezien zijn block-level Elementen - Betekent dat ze starten en eindigen op een nieuwe lijn - Goeie manier: om aan hen te denken als dat ze elementen zijn met ellebogen. Ze duwen de elementen boven en onder hen naar een nieuwe lijn - Inline elementen hebben die kracht niet! - Elk element is ofwel een block of inline. Je zal dit binnenkort wel kunnen veranderen met CSS - We gaan nu enkele inline elementen overlopen. Ze worden met name gebruikt om bepaalde accenten te leggen binnen een tekst en duwen wat voorafgaat en volgt niet naar een nieuwe lijn. Ze zitten ook zo goed als steeds binnen een ander element. - We komen er nog op terug want er zijn er vele: vandaag twee belangrijke om ermee te kunnen oefenen

## div ``` < div id =" news" > < h1 > New This Week < p > We've been working on... < p > And last but not least,... ``` - Met een div-element maak je een logische groepering van elementen op de pagina - Je geeft daarmee aan dat ze samen horen of je wil ze samennemen voor gebruik in CSS/JavaScript - Tip: een div met daarin slechts één element is NOOIT nodig. Het is een zonde die ik vaak bij studenten zie - Kwaal: "divitis", veel meer divs gebruiken dan nodig. Je ziet het zelfs bij professionele designers. Wees zuinig op je elementen. Heb je die allemaal wel nodig? - Noot: in het voorbeeld hierboven had ik perfect een section element kunnen gebruiken. Het is waar: sinds HTML5 is de noodzaak om HTML5 te gebruiken serieus verminderd
## span ``` < ul > < li > John: < span class =" tel" > 999.8282 < li > Paul: < span class =" tel" > 888.4889 < li > George: < span class =" tel" > 888.1628 < li > Ringo: < span class =" tel" > 999.3220 ``` - spans hebben dezelfde voordelen als divs - Wordt gebruikt binnen andere elementen - Het is een inline-element en kan enkel andere inline elementen bevatten - Gebruik hangt vast aan verder gebruik door CSS/JavaScript

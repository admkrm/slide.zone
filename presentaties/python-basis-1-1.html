## De terminal en hoe ze in orde krijgen
## De terminal - Onze terminal is eigenlijk een terminal emulator - Met de terminal bedoelen we de toepassing die je uitvoert om toegang te krijgen tot de shell - De shell geeft ons de commando's die we via een command-line interface kunnen gebruiken - Werk je met __Mac of Linux__ dan is de terminal standaard ingebouwd in je besturingssysteem - Je start in dat geval de terminal-toepassing en je kan ze meteen beginnen te gebruiken - Er zijn verschillende alternatieve terminal-toepassingen die je kan installeren (bvb [iterm2](https://iterm2.com/) op Mac)
## De terminal onder Windows - Op __Windows__ heb je standaard toegang tot command-line toepassingen zoals de [Command Prompt](https://en.wikipedia.org/wiki/Cmd.exe) - Het is evenwel beter om de Windows terminal-toepassing te gebruiken - Bij Windows 11+ is de kans groot dat deze toepassing standaard geïnstalleerd is - Indien niet: download het van de [Microsoft Store](https://apps.microsoft.com/store/detail/windows-terminal/9N0DX20HK701) of van de [GitHub repository](https://github.com/microsoft/terminal) - Hier vind je een volledige en goeie tutorial om [Windows terminal te installeren](https://realpython.com/python-coding-setup-windows/#installing-windows-terminal) - Na het installeren van Windows terminal vind je de terminal-toepassing in het startmenu onder terminal
## De terminal en de verschillende shells - Afhankelijk van je besturingssysteem voer je je terminal commando's uit in een specifieke shell - Voor Linux is dit quasi steeds [Bash](https://en.wikipedia.org/wiki/Bash_(Unix_shell)), bij MacOS is dat [Zsh](https://en.wikipedia.org/wiki/Z_shell) - Windows terminal werkt standaard met [PowerShell](https://learn.microsoft.com/en-us/powershell/) - Deze shells verschillen voor wat bepaalde commando's betreft, maar ze delen de meeste basiscommando's - Voor een Bash shell (zoals Linux) volg je [deze](https://itsfoss.com/install-bash-on-windows/) richtlijnen - Je kan ook Git (inclusief de Git Bash shell) [installeren](https://git-scm.com/)
## De meest gebruikte terminal commando's
## De meest gebruikte terminal commando's - Onder terminal commando's verstaan we de instructies die je ingeeft om specifieke taken uit te voeren - Wij concentreren ons hier op het werken met het bestandssysteem - Daarnaast en daaraan gekoppeld bekijken we hoe je bestanden en directories aanmaakt en wijzigt - Onder bestandssysteem verstaan we de hiërarchisch geordende structuur van bestanden en directories - Je bent gewoon het te bekijken en te wijzigen vanuit een GUI-toepassing zoals Windows Explorer
## Het bestandssysteem: pwd en ls - Met pwd of "print working directory" krijg je je huidige plaats binnen het bestandssysteem te zien ```Bash $ pwd ``` - Met ls of "list" krijg je te zien welke bestanden en directories je "working directory" bevat ```Bash $ ls ```
## Het bestandssysteem: ls met flags - Een command-line flag (ook gekend als option of switch) wijzigt de werking van een commando - Welke flags je kan gebruiken hangt af van commando tot commando - Bij ls heb je de -a flag. Deze staat voor _all_, dus ook verborgen bestanden (met een puntje voor de naam) - Er is ook de -l flag. Deze betekent _long_ en levert gedetailleerde informatie af over de bestanden en folders - Je krijgt dan ook informatie over permissies (owner en group). Meer info [hier](https://en.wikipedia.org/wiki/File-system_permissions#Notation_of_traditional_Unix_permissions) - alle mogelijke opties: _man ls_. Je kan elke flag individueel gebruiken of combineren zoals hier: ```Bash $ ls -al ```
## Environment variabelen en PATH ```Bash # Linux + macOS $ echo "${PATH//:/\n}" # Windows PS> (cat ENV:Path) -Split ";" ``` - Environment variables are variables that store stateful information about the environment in which the terminal is running. They can be used to store information such as the current working directory, the location of installed software, or the user’s home directory. The terminal can access and use this information to determine how to operate and where to look for files. - PATH is an environment variable that stores a list of directories. To see what paths are in your PATH, call the following command: - When you enter a command in the terminal, the system will look for a program that matches the command in the directories listed in the PATH. The list visible after running the previous command is the list of locations that your system will look for when evaluating which program to run.
## Navigeren door je bestandssysteem - We gebruiken het commando _cd_ - staat voor _change directory_ - om onze _working directory_ te wijzigen - We gebruiken de naam van de directory gelegen in de huidige directory en gaan zo een niveau dieper - We noemen dit navigeren aan de hand van een relatief pad ```Bash $ pwd /Users/kristofmichiels $ cd ap $ pwd /Users/kristofmichiels/ap ```
## Navigeren door je bestandssysteem - Je kan ook navigeren aan de hand van een volledig of absoluut pad, zoals hier te zien: ```Bash # Linux + macOS cd /Users/kristofmichiels/ap # Windows PS> cd C:\Users\\Desktop ```
## Navigeren door je bestandssysteem - Je kan ook een directory terug naar boven gaan en doet dit met _cd .._ - Dit commando laat je één niveau opschuiven in de hiërarchie van het bestandssysteem - De twee puntjes (..) vertegenwoordigen de parent directory van de huidige - Trouwens: met _cd_ (dus zonder de puntjes) ga je, waar je je ook mag bevinden, automatisch terug naar de start-directory (meestal de user-folder) ```Bash $ pwd /Users/kristofmichiels/ap $ cd .. $ pwd /Users/kristofmichiels ```
## Directories aanmaken of verwijderen - De volgende commando's laten je toe om op de snelst mogelijke wijze projectconfiguraties aan te maken - Met het _mkdir_ commando maak je een nieuwe folder aan binnen de huidige _working directory_ ```Bash $ mkdir mijn_python_project ``` - met _rmdir_ kan je een lege directory terug verwijderen. Een niet-lege directory verwijder je met _rmdir -r_ ```Bash $ rmdir mijn_python_project ```
## Bestanden aanmaken of verwijderen - Met het touch/ni commando maak je een leeg bestand aan met een meegegeven naam - Indien het bestand al bestaat, wordt de timestamp (voor toegang en aanpassing) geactualiseerd ```Bash # Linux + macOS $ touch main.py # Windows PS> ni main.py ``` - Een bestand verwijderen doe je met _rm_: ```Bash $ rm main.py ```
## Bestanden aanmaken of verwijderen - Met echo kan je via de standard output stream (stdout) een string toevoegen aan een bestand - Standaard wordt de stdout op het scherm getoond maar je kan deze ook redirecten, bvb naar een bestand - We redirecten deze output met het > symbool (= "caret symbol") - Wanneer je redirect naar een niet-bestaand bestand dan wordt dit bestand voor jou aangemaakt ```Bash $ echo 'print("Hallo wereld!")' > main.py ```
## Inhoud van bestanden weergeven met cat en tail - _cat_ laat toe de inhoud van een bestand af te drukken. Goed voor minder omvangrijke bestanden - Het commando is is afgeleid van concatenate (samenvoegen): je kan ook meerdere bestanden meegeven als argument en de inhoud van deze bestanden wordt achter elkaar weergegeven - _tail_ drukt de laatste N aantal lijnen (de staart) van een bestand af. Standaard is N = 10. In het vb hieronder laten we de laatste 100 lijnen afdrukken - Je kan met _tail -f_ (=follow) de veranderingen laten weergeven aan de inhoud van een bestand - Het verdient in deze context ook de aanbeveling om jezelf vertrouwd te maken met [vim](https://www.linkedin.com/learning/learning-vim) of met [neovim](https://neovim.io/) ```Bash $ cat main.py $ tail -n 100 mijn_bestand.txt ```
## Het terminal venster leegmaken - _clear_ is een vaak gebruikt commando om je terminal venster leeg te maken ```Bash $ clear ```
## Virtuele Python omgevingen
## Virtuele Python-omgevingen - Krachtig packaging systeem om de dependencies van je projecten te beheren - Per project-directory kan je een privé-kopie maken van de interpreter én de gebruikte dependencies - Onder dependencies verstaan we alle software-componenten (zoals externe libraries) die nodig zijn om je project succesvol te kunnen runnen - De virtuele omgeving vormt een volledig geïsoleerde python omgeving binnen de context van een directory - Wij gebruiken de [venv-module van Python](https://docs.python.org/3/library/venv.html)
## Een virtuele omgeving opzetten - Het venv-script wordt meegeleverd met de Python-interpreter - De tweede venv op de lijn is de naam die we kiezen voor onze virtuele Python-omgeving. Je kan een andere kiezen, maar is zowat standaard om "venv" te kiezen - Na uitvoeren van het script wordt een venv folder aangemaakt waarin de interpreter en eventueel ook later geïnstalleerde dependencies wordt opgeslagen ```Bash $ python3 -m venv venv ```
## Een virtuele omgeving activeren - We activeren de venv via een script in de virtuele omgeving - Zichtbaar aan de prompt van je bash/zsh shell dat venv actief is ```Bash # windows: $ venv\Scripts\activate # unix, mac: $ source venv/bin/activate ```
## Je virtuele omgeving testen - Kan je doen met which, dat het pad geeft naar bvb python of pip - Test deze commando's met én zonder een geactiveerde virtuele omgeving en observeer het verschil - Je ziet hier het verschil tussen een globale en lokale omgeving ```Bash $ which python $ which pip ```
## Python installer pip - We gebruiken het met Python meegeleverde pip om externe dependencies te installeren - Pip is een recursief acronym dat staat voor "Pip Installs Packages" of "preferred installer program" :-) - Zorg er voor dat je virtuele omgeving geactiveerd is als je packages installeert - Bekijk de verzameling van gepubliceerde Python packages op PyPI: https://pypi.org/ - Bijvoorbeeld om de requests-library te installeren: ```Bash $ pip install requests ```
## Doorgeven of deployen van een virtual environment - Om de lijst van de geïnstalleerde dependencies te zien: ```Bash $ pip list ``` - Dependencies opslaan in een requirements-bestand ```Bash $ pip freeze > requirements.txt ``` - Een virtuele omgeving configureren op basis van een requirements-bestand ```Bash $ pip install -r requirements.txt ```
## Een virtuele omgeving de-activeren - Hoe keer je terug naar de globale omgeving? - Gebruik "deactivate" in de terminal om de virtuele omgeving te verlaten ```Bash deactivate ```
## Werken met git
## Werken met git - Git is een versie-beheersysteem dat wijzigingen in je projectstructuur bijhoudt doorheen de tijdens - Je kan daarbij steeds terugkeren naar een vorige situatie indien nodig - Er zijn grafische tools om te werken met Git: eentje ervan is de source control tab in VSCode - De meest directe manier om met Git te interageren is evenwel via de terminal - Je installeert git via https://git-scm.com/downloads
## Een git repository initiëren en de status bekijken - The first step in using Git is to initialize a repository. A repository is a container that holds all your project files, folders, and metadata. - When you run the git init command, Git creates an empty repository in the current working directory. This creates a new subdirectory named .git/ that contains all of the necessary repository files. ```Bash $ git init ``` - The git status command shows your repository’s current status. It displays which branch you’re on and whether or not there are any commits. ```Bash $ git status ```
## .gitignore - You can ignore files in Git by creating a .gitignore file and listing the files and folders that you want to ignore in that file: - https://github.com/github/gitignore/blob/main/Python.gitignore ```Bash $ echo "venv" > .gitignore ```
## Track Files With Git - You use the git add command to tell Git which files you want to track - The dot refers to the current directory. Using . at the end of the git add command tells Git to track all the files in the current directory ```Bash $ git add . ```
## commits - With the git commit command, you take a snapshot of the current state of your files and store it in the repository’s history. The -m flag allows you to include a message describing your changes. The output shows the branch that you’re on and the number of files changed. - After you commit any changes, it’s a good idea to check the status of your Git repository again: ```Bash $ git commit -m "Initiate git repository" $ git status ``` https://realpython.com/python-git-github-intro/
